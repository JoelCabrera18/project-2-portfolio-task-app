## 1. Arquitectura

El sistema de traducciones se ejecuta completamente en **tiempo de ejecución (Runtime)** para ofrecer un cambio de idioma instantáneo. El flujo es el siguiente:

```
┌──────────────────────────────────────────────────────────┐
│                      Settings Page                       │
│  (El usuario hace clic en el botón de cambiar idioma)    │
└──────────────┬───────────────────────────────────────────┘
               │ setLanguage('en' | 'es') → updateSetting('language', lang)
               ▼
┌──────────────────────────────────────────────────────────┐
│                   AuthService                            │
│  settings = signal<UserSettings>(...)                    │
│                                                          │
│  effect(() => {                                          │
│    // Detecta cambio de lenguaje                         │
│    translationService.setLanguage(lang)                  │
│  })                                                      │
└──────────────┬───────────────────────────────────────────┘
               │ Descarga asíncronamente el archivo JSON
               ▼
┌──────────────────────────────────────────────────────────┐
│                 TranslationService                       │
│  translations = signal<Record<string, any>>(...)         │
│  currentLang = signal<'es' | 'en'>                       │
└──────────────┬───────────────────────────────────────────┘
               │ Emite el nuevo diccionario al pipe
               ▼
┌──────────────────────────────────────────────────────────┐
│                    TranslatePipe                         │
│  Recibe las llaves en los HTML y las traduce             │
│  reactivamente sin recargar la página.                   │
└──────────────────────────────────────────────────────────┘
```

## 2. Estructura de Archivos JSON

Los archivos de traducción se encuentran en el directorio `public/i18n/` y se copian automáticamente al directorio raíz del servidor:

- `public/i18n/es.json` (Español)
- `public/i18n/en.json` (Inglés)

### Reglas de estructura:
* Usa nombres de llaves jerárquicos (anidados) agrupados por módulo o funcionalidad.
* Ambas traducciones deben tener exactamente la misma estructura de llaves.

**Ejemplo de llave en `es.json`:**
```json
{
  "auth": {
    "login": {
      "title": "Iniciar sesión",
      "submit": "Entrar"
    }
  }
}
```

## 3. Cómo consumir traducciones en plantillas HTML

Para utilizar traducciones en las plantillas HTML, debes:

1. Importar el `TranslatePipe` en los imports de tu componente.
2. Utilizar el pipe `translate` con un string que represente la ruta de la llave.

### Componente (TypeScript)
```typescript
import { Component } from '@angular/core';
import { TranslatePipe } from '../../shared/pipes/translate.pipe';

@Component({
  selector: 'app-mi-componente',
  standalone: true,
  imports: [TranslatePipe], // <- Importante añadirlo aquí
  templateUrl: './mi-componente.html',
})
export class MiComponente {}
```

### Plantilla (HTML)
```html
<h1>{{ 'auth.login.title' | translate }}</h1>
<button>{{ 'auth.login.submit' | translate }}</button>
```

## 4. Cómo consumir traducciones en componentes TypeScript

Si necesitas traducir textos en el código TypeScript (por ejemplo, notificaciones de éxito/error o validaciones), puedes inyectar directamente el `TranslationService`.

```typescript
import { Component, inject } from '@angular/core';
import { TranslationService } from '../../shared/services/translation.service';
import { NotificationService } from '../../shared/services/notification.service';

@Component({ ... })
export class MiComponente {
  private translationService = inject(TranslationService);
  private notification = inject(NotificationService);

  ejecutarAccion() {
    // Realizar lógica de negocio...
    
    // Obtener traducción dinámica en el código
    const mensajeExito = this.translationService.translate('auth.login.success');
    this.notification.success(mensajeExito);
  }
}
```

### 💡 Traducción automática en Notificaciones
El componente visual de notificaciones (`NotificationComponent`) está suscrito al pipe de traducción. Esto significa que puedes enviar directamente una llave de traducción a través del `NotificationService` y esta se traducirá automáticamente en la pantalla del usuario en tiempo real.

**Ejemplo de uso:**
```typescript
// En tu componente o servicio:
this.notification.success('notifications.loginSuccess');
```
*Si el usuario cambia el idioma mientras la notificación sigue activa en pantalla, el texto de la notificación se traducirá instantáneamente.*

## 5. Guía paso a paso al crear una nueva página

Cuando crees una nueva página en la aplicación (ej: una página de Dashboard):

1. **Identifica los textos estáticos**: Haz una lista de los títulos, subtítulos, etiquetas de botones y descripciones que se mostrarán en la página.
2. **Declara las llaves**: Abre `public/i18n/es.json` y `public/i18n/en.json`, y crea un nuevo bloque para la página.
   ```json
   "dashboard": {
     "welcome": "Bienvenido de vuelta",
     "noTasks": "No tienes tareas pendientes"
   }
   ```
3. **Importa `TranslatePipe`**: En el TypeScript de tu componente de página, añade `TranslatePipe` al arreglo `imports`.
4. **Reemplaza los textos**: En el HTML de tu página, cambia los textos duros por llaves usando el pipe `| translate`.
   ```html
   <h2>{{ 'dashboard.welcome' | translate }}</h2>
   ```

---

## 6. Reglas de oro

1. ⚠️ **Nunca dejes textos duros en el HTML**. Todo texto visible al usuario debe consumirse a través de i18n para mantener la consistencia del multiidioma.
2. ⚠️ **Sincronización de llaves**: Siempre que agregues una llave a `es.json`, agrégala inmediatamente a `en.json` (aunque sea con el valor en inglés temporal) para evitar textos vacíos o fallas de traducción en producción.
3. ⚠️ **Dotted notation**: La ruta de la llave para subniveles se escribe separada por puntos (`settings.tabs.appearance`).
